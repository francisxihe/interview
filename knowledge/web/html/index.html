大量 svg 拖拽渲染速度为什么慢？说一下怎么减少回流？ ​

题解

SVG 的拖拽渲染速度慢可能是由于以下几个原因：

  1. DOM 复杂度高：

SVG 通常涉及大量的 DOM 元素，特别是在包含大量细节的图形中。每个 SVG 元素都是一个 DOM 节点，大量节点意味着更多的计算和内存消耗。

  1. 回流（Reflow）和重绘（Repaint）：

拖拽时，浏览器可能需要频繁地进行回流和重绘来更新界面。回流是指浏览器重新计算页面布局，重绘是指更新视觉属性（如颜色、阴影等）。

  1. JavaScript 性能：如果使用 JavaScript 进行拖拽逻辑，脚本的性能也会影响渲染速度。不优化的 JavaScript 代码可能导致性能瓶颈。

为了减少回流和提高 SVG 的拖拽渲染性能，可以尝试以下策略：

  1. 使用 CSS 变换（Transforms）：

利用 CSS 的变换（如translate）进行移动，而不是修改元素的位置属性（如top、left）。CSS 变换可以减少回流，因为它不会改变 DOM 的布局。

  1. 避免不必要的 DOM 更改：

尽量减少对 DOM 的更改。例如，不要在拖拽过程中频繁添加或删除元素。

  1. 使用层（Layers）：

通过将 SVG 元素置于单独的层（例如使用will-change或transform: translateZ(0)）可以提高性能，因为层的变化不会影响其他元素。

  1. 优化 SVG 内容：

简化 SVG 结构，去除不必要的元素和属性，减少 DOM 节点的数量。

  1. 使用请求动画帧（requestAnimationFrame）：

在 JavaScript 中，使用requestAnimationFrame进行动画或拖拽操作，可以确保在合适的时机更新屏幕，减少不必要的重绘和回流。

  1. 虚拟 DOM 技术：

使用 React、Vue 等框架可以有效地管理 DOM 更改，减少回流。

  1. 节流和防抖：

在处理拖拽事件时使用节流（throttle）或防抖（debounce）技术，以减少触发频率。

Canvas 如何实现一个小球按正弦波移动的效果？ ​

淘天

题解

js 控制动画 ​

在 HTML 文档中定义一个 <canvas> 元素，并初始化其 2D 绘图上下文，设定小球的初始位置、速度、大小等属性。

使用 requestAnimationFrame 或类似方法来创建一个循环，不断更新并重新绘制小球。在每次循环中，根据正弦波的数学公式来更新小球的垂直位置。

html
<!DOCTYPE html>
<html>
  <head>
    <title>正弦波上的小球运动</title>
    <style>
      canvas {
        border: 1px solid black;
      }
    </style>
  </head>
  <body>
    <canvas id="myCanvas" width="800" height="400"></canvas>
    <script>
      var canvas = document.getElementById("myCanvas");
      var ctx = canvas.getContext("2d");

      var x = 0; // 小球的初始水平位置
      var y = canvas.height / 2; // 小球的初始垂直位置
      var amplitude = 50; // 正弦波的振幅
      var frequency = 0.05; // 正弦波的频率
      var ballRadius = 10; // 小球的半径

      function drawBall() {
        ctx.beginPath();
        ctx.arc(x, y, ballRadius, 0, Math.PI * 2);
        ctx.fillStyle = "blue";
        ctx.fill();
        ctx.closePath();
      }

      function animate() {
        ctx.clearRect(0, 0, canvas.width, canvas.height); // 清除画布
        drawBall(); // 绘制小球
        x += 2; // 增加小球的水平位置
        y = canvas.height / 2 + amplitude * Math.sin(frequency * x); // 计算新的垂直位置

        if (x > canvas.width) {
          x = 0; // 如果小球超出画布，则重置其位置
        }

        requestAnimationFrame(animate); // 递归调用函数自身，形成动画循环
      }

      animate(); // 启动动画
    </script>
  </body>
</html>

css 动画 ​

使用 CSS transform 结合关键帧动画（keyframes）可以更精细地控制动画，从而更好地模拟小球沿正弦波路径的运动。这种方法可以通过 translateX 和 translateY 函数来分别控制小球在水平和垂直方向上的移动。

html
<!DOCTYPE html>
<html>
  <head>
    <title>正弦波上的小球运动（CSS Transform）</title>
    <link rel="stylesheet" type="text/css" />
    <style type="text/css">
      #ball {
        top: 100px;
        left: 100px;
        width: 20px; /* 小球的大小 */
        height: 20px;
        background-color: blue;
        border-radius: 50%; /* 圆形 */
        position: absolute;
        /* 应用动画，5秒内无限重复，线性速度 */
        animation: moveBall 5s infinite linear;
      }

      @keyframes moveBall {
        0% {
          transform: translateX(0) translateY(0); /* 初始位置 */
        }
        25% {
          transform: translateX(25%) translateY(-50px); /* 向上移动 */
        }
        50% {
          transform: translateX(50%) translateY(0); /* 中间位置 */
        }
        75% {
          transform: translateX(75%) translateY(50px); /* 向下移动 */
        }
        100% {
          transform: translateX(100%) translateY(0); /* 初始位置 */
        }
      }
    </style>
  </head>
  <body>
    <div id="ball"></div>
  </body>
</html>
